<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| notifications realtime stream → client | RLS owner-filter is the boundary; the explicit user_id channel filter reduces noise |
| client mark-read update → Postgres | notifications_update_own restricts updates to the owner's rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Spoofing | forging a notification to another user's bell | mitigate | client has no INSERT path; create_notification (02-02 trigger) is the sole writer (Phase-1 gate) |
| T-02-17 | Information Disclosure | realtime channel leaking another account's notifications | mitigate | channel keyed on user + RLS owner filter + teardown on user change (Pitfall 6) |
| T-02-18 | Tampering | marking another user's notification read | mitigate | notifications_update_own RLS restricts UPDATE to owner rows |
</threat_model>
