<objective>
Build the FND-03 audit-log keystone: an append-only `admin_audit_log` table and a generic `log_admin_action()` SECURITY DEFINER RPC with one signature for every future admin mutation. No admin features write to it this phase (D-11/D-12) — Phase 2 wires the writes and Phase 2 builds the viewer (ADMIN-01/02). This plan only builds the substrate so admin features compose onto it without a schema change.

Purpose: Every future admin action (delete ad, change role, verify, change prices) serializes its before/after into one `details jsonb` via one RPC, recorded in a tamper-evident append-only log.
Output: A new self-contained `cabofeira/supabase/admin_audit_log.sql`.
</objective>
