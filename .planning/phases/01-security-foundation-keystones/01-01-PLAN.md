<objective>
Close the three highest-severity RLS holes on `profiles` and `products` (SEC-01/02/03) and add the `products.status` keystone column with an RLS-enforced "only active is public" SELECT policy (FND-01). This is the core security-boundary work of the phase: the enforcement lives in Postgres (RLS policies + BEFORE-trigger guards), because there is no server-side API between the React app and the table — a direct supabase-js/curl call must be blocked at the DB.

Purpose: A normal authenticated account calling the API directly cannot escalate privileges, steal a listing, mass-assign privileged columns, or read non-active listings of other sellers.
Output: A new `security_guards.sql` (trigger guards), edits to `schema.sql` (status column, SELECT policy rewrite, WITH CHECK additions, demo-credential block removed), and an optional one-line forward-looking expose of `status` in `ProductsContext.jsx`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client(supabase-js/curl) → Postgres | A normal authenticated JWT can call the DB directly; no server-side API mediates. RLS + triggers are the only enforcement. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | `update profiles set role/verified` | mitigate | guard_profiles_update BEFORE UPDATE pins role/verified for non-admins; `with check (auth.uid()=id)` reinforces (SEC-01 / D-16) |
| T-01-02 | Tampering | `update products set seller_id` (listing theft) | mitigate | guard_products_update pins seller_id; `with check (auth.uid()=seller_id)` blocks reassignment (SEC-02 / D-17). NOTE: the admin exemption in `guard_products_update` intentionally allows admins to change `seller_id`/`featured`/etc.; admin-account compromise is therefore an accepted residual risk deferred to Phase 4 (SEC-09). |
| T-01-03 | Tampering | mass-assign featured/seller_verified/views on insert | mitigate | guard_products_insert forces safe defaults for non-admins (SEC-03 / D-18) |
| T-01-04 | Tampering | arbitrary `views` rewrite | mitigate | views guard allows only `old.views+1`; any other value pinned (D-20-safe) |
| T-01-05 | Information Disclosure | direct SELECT of non-active / others' listings | mitigate | SELECT policy `status='active' or owner or admin` (FND-01 / D-01) |
| T-01-06 | Elevation of Privilege | search_path hijack on SECURITY DEFINER guards | mitigate | every guard fn pinned `set search_path = public` |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan (SQL + JS edit only); audit table N/A per RESEARCH |
</threat_model>
